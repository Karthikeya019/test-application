POST status update:
Invoke-RestMethod -Uri "http://127.0.0.1:8001/webhook/status" -Method Post -ContentType "application/json" -Body '{"service_id":"jira","status":"down"}'

- Use curl.exe for native curl syntax if needed.

  GIT WORKFLOW NOTES (in order, with what each does)
=====================================================

--- ONE-TIME SETUP (per repo) ---
git init                          # turn current folder into a Git repo (creates .git)
git config user.name "Name"       # who commits show up as
git config user.email "x@y.com"   # email tied to commits
git remote add origin <repo-url>  # link local repo to the GitHub repo

--- THE CORE LOOP (repeat every time you make changes) ---
git status                        # see what changed / what's staged (ALWAYS run first)
git add .                         # stage ALL changes for the next commit
git commit -m "message"           # snapshot staged changes locally
git push                          # upload commits to GitHub

--- FIRST PUSH OF A BRANCH ---
git branch -M main                # rename current branch to 'main' (match GitHub default)
git push -u origin main           # push AND set tracking (future pushes = just 'git push')

--- WHEN PUSH IS REJECTED (remote has commits you don't) ---
git pull origin main                              # fetch + merge remote changes first
git pull origin main --allow-unrelated-histories  # same, when histories started separately
# (if a merge editor opens: save & close to finish the merge)
git push                                          # now push succeeds

--- STOP TRACKING A FILE (but keep it on disk) ---
git rm --cached <file>            # remove from repo, file stays locally
# then add <file> to .gitignore so it won't come back
git add .gitignore
git commit -m "stop tracking <file>"
git push

--- FIX: EMBEDDED REPO ("adding embedded git repository") ---
# happens when a subfolder has its own .git inside the main repo
Remove-Item -Recurse -Force <subfolder>\.git   # delete inner .git (code is safe)
git rm --cached -f <subfolder>                  # force-remove the stale pointer
git add .                                       # re-add the REAL files
git status                                       # confirm: individual files, not one line

--- .gitignore RULES (concept) ---
# - only affects UNtracked files; already-committed files need 'git rm --cached' first
# - scope = its own folder and everything below it
# - root-level files (outside subfolders) need a ROOT .gitignore
# typical contents:
#   venv/  node_modules/  __pycache__/  .next/  .env  .env.local  *.pyc

--- USEFUL CHECKS ---
git status                        # current state (the source of truth — trust over the GUI)
git log --oneline                 # list past commits, newest first
git branch                        # show branches, * marks current

--- 7) CICD ---
- Frontend CI failed in `npm ci` with `E401: Incorrect or missing password`.
- The failure happened because the workflow was trying to use enterprise npm auth/registry settings that are not available from a personal machine or non-enterprise setup.
- Fix used here: switch the workflow to the generic public npm registry (`https://registry.npmjs.org/`) and remove the unconditional auth step.
- Keep the enterprise Artifactory registry and token lines only as commented reference in the workflow for future enterprise environments.
